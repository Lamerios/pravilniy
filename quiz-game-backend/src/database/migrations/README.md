# Database Migrations

Этот каталог содержит миграции базы данных для Quiz Game приложения.

## Структура миграций

### Основные таблицы (Sprint 1)

1. **001-create-organizations.ts** - Таблица организаций
   - Хранит информацию об организациях
   - Базовая таблица для многопользовательской системы

2. **002-create-users.ts** - Таблица пользователей
   - Хранит информацию о пользователях системы
   - Связана с организациями (многие к одному)
   - Поддерживает роли и права доступа

3. **003-create-game-templates.ts** - Таблица шаблонов игр
   - Хранит шаблоны игр для многократного использования
   - Связана с организациями и пользователями
   - Поддерживает различные типы игр и настройки

## Команды для работы с миграциями

```bash
# Запустить все миграции
npm run migrate:up

# Откатить все миграции
npm run migrate:down

# Сбросить базу данных (откат + миграция)
npm run migrate:reset
```

## Особенности миграций

### Типы данных
- **UUID** - для всех первичных ключей
- **JSONB** - для гибкого хранения настроек и конфигураций
- **ENUM** - для ограниченных наборов значений
- **ARRAY** - для хранения массивов (теги, списки)

### Индексы
- Автоматические индексы на внешние ключи
- Индексы на часто используемые поля для поиска
- GIN индексы для JSONB и ARRAY полей
- Уникальные индексы для полей с ограничением уникальности

### Ограничения целостности
- **CASCADE** для обновлений связанных записей
- **RESTRICT** для предотвращения удаления используемых записей
- **CASCADE** для удаления зависимых записей

### Комментарии
- Все таблицы и поля имеют описательные комментарии
- Комментарии на русском языке для лучшего понимания

### Игровые таблицы (Sprint 1)

4. **004-create-games.ts** - Таблица игровых сессий
   - Хранит информацию об игровых сессиях
   - Связана с организациями, шаблонами и пользователями
   - Поддерживает различные статусы игры

5. **005-create-teams.ts** - Таблица команд
   - Хранит информацию о командах в игре

   - Включает статистику и рейтинги

6. **006-create-rounds.ts** - Таблица раундов
   - Хранит информацию о раундах игры
   - Поддерживает различные типы раундов

7. **007-create-answers.ts** - Таблица ответов
   - Хранит ответы команд на вопросы

   - Включает бонусы и штрафы

## Бизнес-логика системы

### Основной процесс игры
1. **Организация** создает **шаблоны игр** с базовой информацией
2. **Администратор** создает **игровую сессию** на основе шаблона
3. **Команды** регистрируются на игру
4. **Администратор** создает **раунды** для игры
5. Во время игры команды отвечают на бумаге
6. **Администратор** вносит только **баллы за раунд** для каждой команды
7. Система накапливает **общий счет** команд
8. Формируется итоговая таблица результатов

### Важные особенности
- **Вопросы не хранятся в системе** - они существуют только на бумаге
- **Ответы команд не хранятся** - только итоговые баллы за раунд
- **Администратор** контролирует весь процесс игры
- **Администратор** вносит только баллы за раунд для каждой команды
- **Система автоматически** суммирует баллы и формирует рейтинги

## Рекомендации

1. **Всегда создавайте резервную копию** перед запуском миграций в production
2. **Тестируйте миграции** на копии production данных
3. **Используйте транзакции** для атомарности операций
4. **Документируйте изменения** в комментариях к миграциям
5. **Проверяйте откат** миграций перед деплоем

## Отладка

Если миграция не выполняется:

1. Проверьте подключение к базе данных
2. Убедитесь, что база данных существует
3. Проверьте права пользователя БД
4. Посмотрите логи для детальной информации об ошибке

## Структура базы данных

```
organizations (организации)
├── users (пользователи)
│   └── game_templates (шаблоны игр)
└── games (игры)
    ├── teams (команды)
    │   ├── answers (ответы)
    │   └── scores (очки)
    └── rounds (раунды)
        ├── answers (ответы)
        └── scores (очки)
```

### Связи между таблицами

- **organizations** → **users** (1:N)
- **organizations** → **game_templates** (1:N)
- **organizations** → **games** (1:N)
- **users** → **game_templates** (1:N, createdBy)
- **users** → **games** (1:N, createdBy)
- **game_templates** → **games** (1:N)
- **games** → **teams** (1:N)
- **games** → **rounds** (1:N)
- **teams** → **answers** (1:N)
- **rounds** → **answers** (1:N)
- **teams** → **scores** (1:N)
- **answers** → **scores** (1:1, опционально)
