# Database Management Scripts

Полная система управления базой данных для Quiz Game проекта.

## 📁 Структура

```
database/
├── migrations/          # Миграции схемы БД
├── seeders/            # Сиды с тестовыми данными
├── init.ts             # Инициализация БД
├── migrate.ts          # Управление миграциями
├── seed.ts             # Управление сидами
├── setup.ts            # Полная настройка БД
├── reset.ts            # Полный сброс БД
├── clean.ts            # Очистка данных
├── check.ts            # Проверка состояния
├── utils.ts            # Утилиты
└── README.md          # Документация
```

## 🚀 Быстрый старт

### Первичная настройка

```bash
# Полная настройка БД (рекомендуется для новых проектов)
npm run db:setup

# Проверка состояния
npm run db:check
```

### Ежедневная разработка

```bash
# Проверить состояние БД
npm run db:check

# Очистить данные и добавить базовые
npm run db:clean:reseed

# Полный сброс (при проблемах)
npm run db:reset:force
```

## 📋 Доступные команды

### 🔧 Основные команды

| Команда | Описание |
|---------|----------|
| `npm run db:setup` | Полная настройка БД (миграции + все сиды) |
| `npm run db:check` | Проверка состояния БД |
| `npm run db:reset` | Полный сброс БД с подтверждением |
| `npm run db:clean` | Очистка данных с сохранением структуры |

### 🎯 Детальные команды

#### Настройка БД
```bash
npm run db:setup           # Полная настройка (миграции + все сиды)
npm run db:setup:basic     # Настройка с базовыми данными
npm run db:setup:demo      # Настройка с демо игрой
npm run db:setup:full      # Полная настройка (аналог db:setup)
```

#### Сброс БД
```bash
npm run db:reset           # Полный сброс с подтверждением
npm run db:reset:force     # Принудительный сброс без подтверждения
```

#### Очистка данных
```bash
npm run db:clean           # Очистка данных с подтверждением
npm run db:clean:force     # Принудительная очистка
npm run db:clean:reseed    # Очистка + базовые данные
```

#### Миграции
```bash
npm run migrate:up         # Применить миграции
npm run migrate:down       # Откатить последнюю миграцию
npm run migrate:reset      # Сбросить все миграции
```

#### Сиды
```bash
npm run db:seed            # Базовые сиды
npm run db:seed:basic      # Только базовые данные
npm run db:seed:demo       # Только демо игра
npm run db:seed:full       # Все сиды
```

## 📊 Что делают скрипты

### 🔧 `db:setup` - Настройка БД

**Процесс:**
1. ✅ Проверка подключения к БД
2. 📊 Проверка существующих таблиц
3. 🔄 Применение миграций (если нужно)
4. 🌱 Запуск сидеров
5. 🔍 Проверка финального состояния

**Результат:**
- Готовая к работе БД
- Базовые данные (организация, админ, шаблон)
- Демо игра с результатами (опционально)

### 🔄 `db:reset` - Сброс БД

**Процесс:**
1. ⚠️ Запрос подтверждения (если не --force)
2. ✅ Проверка подключения
3. 🗑️ Удаление всех таблиц
4. 📊 Применение всех миграций
5. 🌱 Запуск всех сидеров
6. 🔍 Проверка состояния

**Результат:**
- Полностью чистая БД
- Актуальная схема
- Свежие тестовые данные

### 🧹 `db:clean` - Очистка данных

**Процесс:**
1. ⚠️ Запрос подтверждения
2. ✅ Проверка подключения
3. 📊 Подсчет текущих записей
4. 🧹 Очистка всех таблиц
5. 📊 Проверка результата
6. 🌱 Опциональное пересеивание

**Результат:**
- Пустые таблицы
- Сохраненная структура
- Опциональные базовые данные

### 🔍 `db:check` - Проверка состояния

**Показывает:**
- 🔗 Статус подключения
- 📊 Информация о БД
- 📋 Список таблиц
- 📈 Количество записей в каждой таблице
- ⚠️ Отсутствующие таблицы

## 🎯 Сценарии использования

### Новый разработчик

```bash
# Клонировал репозиторий, настраиваю БД
npm run db:setup

# Проверяю что все работает
npm run db:check
```

### Ежедневная разработка

```bash
# Начало дня - проверяю состояние
npm run db:check

# Нужны чистые данные для тестов
npm run db:clean:reseed

# Тестирую с демо данными
npm run db:seed:demo
```

### Отладка проблем

```bash
# Что-то сломалось, полный сброс
npm run db:reset:force

# Проблемы с миграциями
npm run migrate:reset
npm run migrate:up

# Нужны только базовые данные
npm run db:clean:force
npm run db:seed:basic
```

### CI/CD pipeline

```bash
# В тестах - быстрая настройка
NODE_ENV=test npm run db:setup:basic

# В production - только миграции
NODE_ENV=production npm run migrate:up
```

## ⚙️ Конфигурация

### Переменные окружения

```bash
# .env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=quiz_game_dev
DB_USER=quiz_user
DB_PASSWORD=dev_password

# Для автоматического подтверждения
NODE_ENV=production  # или CI=true
```

### Режимы работы

- **Development**: Запрашивает подтверждение для опасных операций
- **Production**: Автоматически подтверждает операции
- **CI**: Автоматически подтверждает операции

## 🚨 Безопасность

### Подтверждения

Скрипты запрашивают подтверждение для опасных операций:
- `db:reset` - удаление всех таблиц
- `db:clean` - удаление всех данных

### Принудительный режим

Флаг `--force` пропускает подтверждения:
```bash
npm run db:reset:force    # Сброс без подтверждения
npm run db:clean:force    # Очистка без подтверждения
```

### Production защита

В production среде операции выполняются автоматически, но с предупреждениями.

## 🔧 Отладка

### Проблемы с подключением

```bash
# Проверить подключение
npm run db:check

# Проверить настройки в .env
cat .env | grep DB_

# Проверить PostgreSQL
docker ps | grep postgres
```

### Проблемы с миграциями

```bash
# Проверить состояние миграций
npm run migrate:up

# Сбросить миграции
npm run migrate:reset
```

### Проблемы с данными

```bash
# Посмотреть количество записей
npm run db:check

# Очистить и пересеять
npm run db:clean:reseed

# Полный сброс
npm run db:reset:force
```

## 📈 Мониторинг

### Логи выполнения

Все скрипты выводят подробную информацию:
- ✅ Успешные операции
- ⚠️ Предупреждения
- ❌ Ошибки
- 📊 Статистика
- ⏱️ Время выполнения

### Коды выхода

- `0` - Успешное выполнение
- `1` - Ошибка выполнения
- `130` - Прерывание пользователем (Ctrl+C)
- `143` - Завершение по сигналу
